Vector Embeddings (Qdrant)

384 dimensions
all-MiniLM-L6-v2

Every Memory node gets a vector in Qdrant, synced by UUID. Enables semantic search independent of exact wording.

Payload stored with vector:
content, summary, category, project_id, user_id, valid_from, valid_to, tags, created_at
Search flow:
Query → embed → Qdrant top-K → graph expansion via shared entities → re-rank (0.7 vector + 0.3 graph) → return
Entity embeddings:
Optional. Entity name + description embedded for semantic entity lookup.
